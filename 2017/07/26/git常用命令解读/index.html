###3.进阶github命令

比较工作区和暂存区的代码。

比较暂存区和本地仓库的代码。

比较工作区和本地仓库的代码。

只要还没有commit到仓库的代码，都能进行隐藏，因为有时代码写到一半，还不宜提交，但是有别的分支的紧急bug要处理，就要用到这个，把当前状态挂起，处理完bug，再git stash apply恢复。git stash list显示所有挂起的记录，git stash drop删除最近一条挂起记录，git stash pop直接弹出，就是不仅恢复代码，同时删除最近一条记录，git stash clear清空所有挂起记录。

和merge异曲同工，只不过merge是把其他分支的代码按照后来的顺序合并到主分支，rebase是把其他分支的代码和主分支柔和，按照所有人提交的时间顺序合并。在团队开发中常用merge，因为可以知道哪些提交是来自哪里，区分度好。
