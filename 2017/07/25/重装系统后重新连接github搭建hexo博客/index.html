重装系统后重新连接github搭建hexo博客

虽然过程不难，但系统突然崩溃，这些琐碎的步骤实践起来也颇费功夫，故在此记录，将来恢复会更加快速。

1.更改host

由于天朝坑爹的闭关锁国，要给gitbush单独设置代理，配置地址参见以前的文章。

2.安装git、nodejs、markdown、sublimetext

这些都是必要的环境、编辑工具。

3.生成新的ssh

重装系统，以前匹配github的ssh已不在，要重新配置，前面是生成ssh，后面的email是设置全局用户名，这里不输入email后期提交还要输入，一路回车就行，具体步骤也在之前的文章提及。

4.安装hexo

从这步开始正式进入搭建博客，将github上或者存储介质中备份的老博客clone下来单独放在一边，在磁盘中建立新的博客文件夹，右键git bush：

安装hexo博客系统，这里可能由于版本等各种蜜汁原因警告或报错，不用理会。

5.拷贝旧的文件

其实这步就相当于以前的初始化hexo博客，只不过现在有旧博客就不用重新初始化再改配置文件了，除非以前的博客也灰飞烟灭只剩文章，那样就直接建新的博客了。
具体要拷贝的文件是配置、主题、文章，scaffolds、package、gitignore，前三个必须考，后三个其实是init操作生成的。其他文件都不用考，都是后续操作会生成的文件。

6.安装模块、组件

若有主题要配置，还要安装渲染器，安装方法之前文章有提及。
之后执行hexo g、hexo d即可成功部署。
