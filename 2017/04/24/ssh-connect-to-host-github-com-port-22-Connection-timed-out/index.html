ssh-connect to host github.com port 22-Connection timed out

在上传博客时报错如下：

ssh: connect to host github.com port 22: Connection timed out

尝试安装最新版hexo、git、nodejs，重建博客，依然失败。
尝试普通的git操作，也无法完成，报错如下：

Unknown SSL protocol error in connection to github.com:443

在stackoverflow上给出的最佳答案是使用代理，可是我本身就在使用香港的代理，更换了美国的服务器后，依然报错，奇怪的是，github网站能够登陆，但是在cmd中github.com却ping不通。
继续google，原来gitbash本身需要代理。
所以更改host文件，首先右键-属性，给user最高权限，而后再文件末尾加上：

192.30.253.113 github.com
192.30.252.131 github.com
185.31.16.185 github.global.ssl.fastly.net
74.125.237.1 dl-ssl.google.com
173.194.127.200 groups.google.com
192.30.252.131 github.com
185.31.16.185 github.global.ssl.fastly.net
74.125.128.95 ajax.googleapis.com

一通代理，再次ping即可成功。
继续上传博客，提示无法用https提交。
尝试生成ssh秘钥：

ssh-keygen -t rsa

在C:/Users/Administrator/.ssh下生成rsa和rsa-pub文件，拷贝pub中的秘钥到github的ssh中。
修改hexo根目录下config文件，修改deploy的repo中的https地址为git形式，可以在github博客repo中直接找到并切换这两个地址，复制过来：

deploy: 
      type: git
      repository: git@github.com:cwx2810/cwx2810.github.io.git
     branch: master

再尝试hexo g、hexo d，成功上传博客。
